## 6. 학습 및 회고

### 6.1 잘한 점 (Keep)

✅ **상세한 기획/설계 문서**
- 기획서와 설계서가 상세하여 개발 시 참고하기 좋았음
- 데이터 모델, API 명세, UI 와이어프레임이 명확했음

✅ **TypeScript + zod 검증**
- 타입 안전성으로 버그를 초기에 발견
- zod 스키마로 클라이언트/서버 양측 검증 자동화

✅ **Mock 데이터 기반 빠른 개발**
- bkend.ai 연동 전에 Mock 데이터로 전체 기능을 빠르게 구현
- 기능 테스트가 용이했음

✅ **컴포넌트 기반 구조**
- 공통 UI 컴포넌트 분리로 재사용성 증대
- 페이지 조합이 용이했음

✅ **SEO 및 반응형 설계**
- Next.js의 동적 메타데이터로 SEO 구현
- Tailwind CSS로 모바일 우선 반응형 디자인

### 6.2 개선할 점 (Problem)

⚠️ **모듈화 수준 부족**
- hooks (useJobs, useJob) 분리 미구현
- services 레이어 미분리 (API 호출이 컴포넌트 내 인라인)
- 점차 증가할 코드량에 대비 필요

⚠️ **bkend.ai 연동 지연**
- 현재 Mock 데이터 기반이므로 실제 DB 없음
- 하루 2건 제한이 메모리 기반 (재시작 시 리셋)
- 비밀번호 해싱, JWT 서명 미구현

⚠️ **부분 구현된 기능**
- 내 공고 상태 필터 미구현 (FR-11)
- 하루 2건 제한이 불완전 (FR-10)

⚠️ **일부 UI 컴포넌트 미구현**
- Modal, Card 컴포넌트 미구현 (confirm() / 기본 div로 대체)
- VipSlider 미구현 (Grid로 대체)

### 6.3 다음에 시도할 것 (Try)

💡 **bkend.ai BaaS 즉시 연동**
- Mock 데이터 → 실제 DB로 전환
- JWT 기반 인증으로 업그레이드
- 하루 2건 제한을 DB 기반으로 변경

💡 **서비스 레이어 분리**
- `src/services/` 디렉토리 추가
- API 통신 로직을 컴포넌트에서 분리
- 재사용 가능한 API 클라이언트 함수 정의

💡 **Custom Hooks 추출**
- `useJobs()`, `useJob()`, `useJobMutation()` 분리
- 필터 상태를 Zustand (jobFilterStore) 로 관리

💡 **테스트 추가**
- Jest + React Testing Library로 단위 테스트
- E2E 테스트 (Playwright/Cypress)

💡 **관리자 기능 모듈화**
- `src/features/admin/` 디렉토리 분리
- 업체 관리 로직을 `src/features/company/` 로 분리
